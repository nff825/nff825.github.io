网站更新操作手册
办公 2026/4/24

这篇文档用于记录个人网站的标准更新流程，按步骤执行即可完成发布。

一、常用命令

npm run clean

清理 Hexo 缓存和历史生成内容。

npm run build

生成静态网页到 public/。

npm run deploy

将 public/ 内容发布到 GitHub Pages 仓库。

npm run preview

本地预览（清理 + 生成 + 本地服务）。

npm run check:paths

检查 Markdown 是否包含本机绝对路径（如 C:\...），避免图片线上失效。

二、日常发文流程

  1. 新建文章
    npx hexo new post "文章标题"
  2. 编辑文章
    在 source/_posts/ 中编辑对应 .md 文件。
  3. 插入图片
    将图片放在文章同名文件夹中，并使用相对路径，如：![示例](image.png)。
  4. 检查路径
    npm run check:paths
  5. 本地生成
    npm run build
  6. 发布上线
    npm run deploy

三、快速发布流程（推荐）

npm run clean
npm run build
npm run deploy

四、常见问题排查

1. 执行到 Validating config 后不继续

优先检查是否有路径检查报错：

npm run check:paths

若提示类似 C:\Users\...，请改为相对路径，并把图片放入文章资源目录。

2. Permission denied (publickey)

说明在使用 SSH 推送但本机 SSH Key 未配置好。可改为 HTTPS 仓库地址，或配置 GitHub SSH Key。

3. could not read Username for 'https://github.com'

说明 Git 凭据未正确配置。可执行：

git config --global credential.helper manager-core

然后重新 npm run deploy，按提示登录 GitHub。

4. 部署后网页没变化

  1. 确认 npm run deploy 最终成功。
  2. 等待 1-5 分钟再刷新。
  3. 浏览器强制刷新（Ctrl + F5）。

五、建议

每次发布前至少执行一次 npm run check:paths，可以提前避免绝大多数“本地正常、线上图片丢失”的问题。

1 静力分析

1.1 含义

计算在固定不变的载荷作用下结构的响应，不需要考虑惯性 (inertia) 和阻尼 (damping) 的影响

1.2 公式

$$F=[K][u]$$

  • K 是结构刚度 (stiffness)，$K=AE/L$
  • A 是横截面面积，E 是杨氏模量，L 是长度

1.3 理论计算结果

应力

$$\sigma=F/A$$

位移
$$d=PL/AE$$

能量有限元(MATLAB)
2025/4/30

主程序

功能

指定参数空间（激励的位置、幅值、频率），进行能量有限元分析

代码

%% 获取数据 不同激励设置
clear; close all; clc
%% ==========创建数据存储文件夹 ==========
output_folder = 'EFEA_data_20250429_excitation_1800';
if ~exist(output_folder, 'dir')
    mkdir(output_folder);
end
%% ========== 设置固定参数 ==========
geometry.length_x = 0.5075;    % 薄板在 x 方向的长度
geometry.length_y = 0.4375;    % 薄板在 y 方向的长度
geometry.h = 42.4382e-3;       % 薄板的厚度

num_elements_x = 30;            % x 方向的单元数量
num_elements_y = 30;            % y 方向的单元数量
num_nodes = (num_elements_x+1)^2; % 节点数量

material.E = 1.6966e9;          % 弹性模量
material.rho = 94.2547;         % 密度
material.mu = 0.3;              % 泊松比
material.damping_ratio = 0.05; % 阻尼比
%% ========== 设置可变参数范围 ==========
frequencies = [500:10:550];
amplitudes = [1:0.1:1.5];
excitation_shifts = generate_excitation_shifts(num_nodes, 50);
COUNT = numel(frequencies) * numel(amplitudes) * numel(excitation_shifts);
%% ========== 生成数据 (并行计算) ==========
delete(gcp); % 关闭并行池
CoreNum = 4; % 设定机器CPU核心数量
parpool(CoreNum); % 开启并行池
parfor counter = 1:COUNT
    [frequency, F, offset] = get_parameters(counter, frequencies, amplitudes, excitation_shifts);

    %% ========== 激励设置 ==========
    excitation_freq = frequency;                    % 激励频率
    excitation_amp = F;                              % 激励幅值
    excitation_pos = round(num_nodes/2) + offset;    % 激励位置
    %% ========== 能量有限元分析 ==========
    output_filename = sprintf('%s/data_%d.txt', output_folder, counter);
    energy_density = EFEM_2D_plate(geometry, material, struct('freq', excitation_freq, 'amp', excitation_amp, 'pos', excitation_pos), num_elements_x, num_elements_y);
    %% ========== 写入参数数据 ==========
    fileID = fopen(output_filename, 'w');
    fprintf(fileID, '%.6f %.6f %.6f %.6f %.6f %.6f %.6f %.6f %.6f %.6f\n', ...
        excitation_freq, excitation_amp, excitation_pos, ...
        material.E, material.rho, material.mu, material.damping_ratio, ...
        geometry.length_x, geometry.length_y, geometry.h);
    %% ========== 换算能量密度（dB） ==========
    E = reshape(energy_density, num_elements_x+1, num_elements_y+1);
    E_ref = 1e-12;
    E(E < E_ref) = E_ref;
    E_dB = 20 * log10(E / E_ref);
    %% ========== 写入能量密度数据 ==========
    [x_coords, y_coords] = meshgrid(linspace(0, geometry.length_x, num_elements_x+1), linspace(0, geometry.length_y, num_elements_y+1));
    for i = 1:numel(x_coords)
        fprintf(fileID, '%.6f %.6f %.6f\n', x_coords(i), y_coords(i), E_dB(i));
    end
    fclose(fileID);
    fprintf('%d/%d | freq = %d amp = %d offset = %d | %s\n', counter, COUNT, frequency, F, offset, output_filename);
end

disp('done');

蚁群算法

1. 概述

蚂蚁

  • 在其经过的路径上释放信息素
  • 能够感知其它蚂蚁释放的信息素
  • 以较大的概率优先选择信息素浓度较高的路径

信息素

  • 其浓度表征路径的远近
  • 会随时间推移而逐渐衰减

解决优化问题的基本思路

  • 蚂蚁的行走路径——可行解
  • 整个蚂蚁群体的所有路径——解空间
  • 最终，整个蚂蚁群体会在正反馈的作用下集中在最佳路径上——最优解

2. 数学模型

3. 算法流程

  1. 初始化参数
    • 蚁群规模
    • 信息素重要程度因子
    • 启发函数重要程度因子
    • 信息素挥发因子
    • 信息素释放总量
    • 最大迭代次数
    • 迭代次数初值
  2. 构建解空间
    • 将蚂蚁随机置于不同出发点，对每只蚂蚁计算其下一个访问的城市
  3. 更新信息素
    • 计算每条蚂蚁经过的路径长度，记录当前最优解，更新每条路径的信息素浓度
  4. 判断是否终止

4. 特点

  • 正反馈，逼近最优解
  • 个体通过信息素影响环境，个体能实时感知环境变化，个体间通过环境间接通信
  • 多个个体同时进行并行计算，提高算法的计算能力和运行效率
  • 启发式的概率搜索方式，不易陷入局部最优，易于寻找全局最优解

旅行商问题 (Traveling Salesman Problem, TSP)

1. 概述

  • 在 $N$ 个城市中各经历一次后回到出发点，使所经过的路程最短
  • 不考虑方向，可能存在的闭合路径数目为 $\frac{(N-1)!}{2}$
  • TSP 问题常被用于验证算法性能

深度学习——基于梯度的优化
深度学习 2023/10/19

神经网络的“引擎”：基于梯度的优化

对于第一个模型示例，每个神经层都对输入数据进行如下变换：

output = relu(dot(input, W) + b)

在这个表达式中，W 和 b 均为该层的属性，被称为该层的**权重 (weight) **或可训练参数 (trainable parameter)，分别对应属性 kernel 和 bias。这些权重包含模型从训练数据中学到的信息。

一开始，这些权重矩阵取较小的随机值，这一步叫作**随机初始化 (random initialization)。下一步则是根据反馈信号逐步调节这些权重。这个逐步调节的过程叫作训练 (training)**，也就是机器学习中的“学习”过程。

上述过程发生在一个训练循环 (training loop) 内，其具体流程如下。在一个循环中重复下列步骤，直到损失值变得足够小。

​ (1) 抽取训练样本 x 和对应目标 y_true 组成的一个数据批量。

​ (2) 在 x 上运行模型 (这一步叫作前向传播，forward pass)，得到预测值 y_pred。

​ (3) 计算模型在这批数据上的损失值，用于衡量 y_pred 和 y_true 之间的差距。

​ (4) 更新模型的所有权重，以略微减小模型在这批数据上的损失值。

loss_value = f(W)

张量 grad(loss_value, W0) —— f 在 W0 点的导数，其形状与W相同

每个元素 grad(loss_value, W0)[i, j] ——当 W0[i, j] 发生变化时 loss_value 变化的方向和大小

张量 grad(loss_value, W0) ——函数 f(W) = loss_value 在 W0 处的梯度，也叫作 “loss_value 相对于 W 在 W0 附近的梯度”

小批量随机梯度下降 (mini-batch stochastic gradient descent，小批量 SGD)

​ (1) 抽取训练样本 x 和对应目标 y_true 组成的一个数据批量

​ (2) 在 x 上运行模型，得到预测值 y_pred，这一步叫作前向传播

​ (3) 计算模型在这批数据上的损失值，用于衡量 y_pred 和 y_true 之间的差距

​ (4) 计算损失相对于模型参数的梯度，这一步叫作反向传播 (backward pass)

​ (5) 将参数沿着梯度的反方向移动一小步，比如 W -= learning_rate * gradient，从而使这批数据上的损失值减小一些，学习率 (learning_rate) 是一个调节梯度下降“速度”的标量因子

小批量 SGD 的变体

  1. 真 SGD (true SGD)：每次迭代只抽取一个样本和目标
  2. 批量梯度下降 (batch gradient descent)：每次迭代都在所有数据上运行
  3. 优化方法 (optimization method) 或优化器 (optimizer)：
    1. 带动量的 SGD、Adagrad、RMSprop 等
    2. 它们的不同之处在于，计算下一次权重更新时还要考虑上一次权重更新，而不是仅考虑当前的梯度值
    3. 动量解决了 SGD 的两个问题：收敛速度和局部极小值
